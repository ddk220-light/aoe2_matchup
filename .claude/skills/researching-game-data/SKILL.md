# Researching Game Data

## Overview

Verify AoE2 game mechanic values by following a strict source hierarchy: dat file (ground truth) -> extraction pipeline -> database -> simulation code. Never trust a single source.

## Source Hierarchy (follow this order)

1. **Dat file** (ground truth): `extraction/empires2_x2_p1.dat` via genieutils
2. **Extraction pipeline**: `extraction/extract_units.py`, `extract_effects.py`, `extract_advanced.py`
3. **Reference DB**: `webapp/aoe2_reference.db` (generated by `analysis/generate_reference.py`)
4. **Main DB**: `webapp/aoe2_units.db` (generated by `analysis/generate_main_db.py`; legacy — no app route reads it, prefer the reference DB)
5. **Simulation code**: `webapp/simulation.py` (what actually runs)
6. **Online sources**: Wiki, forums, community tests (for context, NOT as ground truth)

## Environment Setup

ALWAYS activate venv before using genieutils:
```bash
source venv/bin/activate
```

genieutils API quirks:
- `dat.civs[0].units[ID]` to access units (NOT `dat.civilizations`)
- Unit attributes: `u.type_50` for combat stats (accuracy, attacks, dispersion)
- Creatable: `u.creatable` (NOT `u.type_70`)
- Unit type field: `u.type` (NOT `u.unit_type`)
- Accuracy: `t50.accuracy_percent` and `t50.accuracy_dispersion`
- Projectile counts: `creatable.total_projectiles`, `creatable.max_total_projectiles`
- Secondary projectile: `creatable.secondary_projectile_unit` (-1 = none)
- Attacks list: `[(a.class_, a.amount) for a in t50.attacks]`

## Research Checklist

1. **Read the dat file first** - Get the raw game engine values for the unit(s)
2. **Trace the extraction pipeline** - How does `extraction/extract_units.py` transform the raw value? Is it extracted at all?
3. **Check the reference DB** - Does the generated value match the dat file?
4. **Check the main DB** - Is the value transferred from reference to main DB? Missing columns?
5. **Check simulation code** - What value does the sim actually use? Hardcoded fallbacks?
6. **Check tech effects** - Do any techs modify the value? Check `extraction/extract_effects.py` attr mappings
7. **Search online** - Cross-reference with wiki, AoEZone, official forums for community understanding
8. **Compare with similar units** - Check the same property on a well-known unit for sanity

## Common Pitfalls

- **Check the projectile unit too**: The projectile unit (type 60) has its own `accuracy_percent` and `type_50` stats. The main unit's accuracy controls the initial scatter, but the projectile object may have different accuracy (e.g., Organ Gun main=0%, projectile 1789=75%).
- **accuracy not in unit_stats table**: `prepare_combat_unit()` defaults to 100 via `.get("accuracy", 100)`. Check if the column exists before trusting it.
- **Extraction only gets main unit accuracy**: `extraction/extract_units.py` line 423 reads `t.accuracy_percent` from the main unit, NOT the projectile unit. Projectile accuracy is silently lost.
- **Scrambled unit names**: dat file names for newer units are wrong (e.g., "FLNCER" for Fire Lancer). Use `internal_name` or numeric ID.
- **blast_damage semantics**: fractional (0.2-0.5) = trample %, 1.0 = siege splash, -5.0 = no trample. Don't assume.
- **attribute 63 is NOT "ignores armor"**: It's a general-purpose attribute used for many things. Keep ability flags hardcoded.
- **secondary_projectile_unit = -1** means all projectiles are primary (e.g., Organ Gun). Different from CKN which has a separate secondary projectile unit.
- **accuracy_percent = 0** does NOT mean "never hits": It means every shot uses the dispersion cone. Projectiles can still hit targets within the scatter area.

## Quick Reference: Key Dat File Fields

| Field | Location | Meaning |
|-------|----------|---------|
| `accuracy_percent` | `type_50` | % chance projectile aims true (0-100) |
| `accuracy_dispersion` | `type_50` | Half-radius of miss scatter cone |
| `total_projectiles` | `creatable` | Number of projectiles fired per volley |
| `secondary_projectile_unit` | `creatable` | ID of secondary proj unit (-1 = all primary) |
| `blast_width` | `type_50` | Splash damage radius (0 = none) |
| `blast_damage` | `type_50` | Splash/trample multiplier |
| `rear_attack_modifier` | `creatable` | HP regen per minute (attr 109) |
| `charge_type` | `creatable` | Special charge mechanic (1=melee, 3=splash, 4=dodge, 5=block, 6=fire proj, 7=burst) |
